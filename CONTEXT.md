# Dastorkon — Глоссарий предметной области

## Основные термины

### Меню / Menu
- **Блюдо (Dish)** — позиция меню, которую можно заказать. Имеет категорию, цену, вес, описание, фотографии и набор характеристик (тегов).
- **Категория меню (Category)** — раздел, группирующий блюда (например, "Салаты", "Горячее"). Имеет название, описание и SVG-иконку.
- **Сборка (Combo)** — набор блюд, продаваемый как единая позиция по фиксированной цене. Имеет свою категорию.
- **Категория сборки (ComboCategory)** — раздел для группировки сборок (например, "Сеты", "Комбо на компанию").
- **Характеристика (Tag)** — атрибут блюда (например, "Острое", "Хит", "Вегетарианское"). Имеет настраиваемый цвет фона и текста для отображения на карточке.

### Заказы / Orders
- **Заказ (Order)** — запрос клиента на приготовление/доставку набора блюд. Содержит информацию о клиенте, составе заказа, статусе обработки.
- **Позиция заказа (OrderItem)** — конкретное блюдо внутри заказа с зафиксированной ценой и количеством.

## Отношения

- Категория → Блюдо (один-ко-многим)
- Категория сборки → Сборка (один-ко-многим)
- Блюдо → Сборка (многие-ко-многим: блюдо может входить в разные сборки)
- Блюдо → Характеристика (многие-ко-многим: блюдо может иметь несколько тегов)
- Заказ → Позиция заказа (один-ко-многим)

## Статусы заказа

- **Новый (new)** — только что создан, требует обработки
- **В обработке (processing)** — принят в работу
- **Завершён (completed)** — выполнен
- **Отменён (cancelled)** — отменён

## Границы контекста

- Меню (core app) — управление каталогом блюд, категорий, сборок, характеристик
- Заказы (orders app) — обработка заказов клиентов, интеграция с Telegram
