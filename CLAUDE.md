# Dastorkon — Multilingual Restaurant Platform

## 🏗 Структура проекта
Проект представляет собой монорепозиторий, разделенный на две изолированные среды:
*   `./frontend/` — Фронтенд на Next.js, TypeScript, Tailwind CSS. Ламповый UI с пергаментными элементами и темно-синим фоном.
*   `./backend/` — Бэкенд на Django, Django REST Framework, база данных SQLite (для локальной разработки)[cite: 3].

## 📂 Специфика Backend-архитектуры
Основная бизнес-логика инкапсулирована в приложении `core`[cite: 3].
*   **Модели меню:** Управление блюдами (`Dish`), категориями (`Category`, `ComboCategory`) и тегами (`Tag`)[cite: 3].
*   **Визуальные теги:** Теги поддерживают настройку внешнего вида на уровне БД (`color_bg`, `color_text`, `opacity`)[cite: 3].
*   **Мультиязычность:** Данные моделей локализованы через `translation.py` (настроена поддержка языков: `en`, `ru`, `kg`/`ky`)[cite: 3].
*   **Медиа:** Иконки категорий и фотографии блюд загружаются в `backend/media/categories/` и `backend/media/dishes/`[cite: 3].

## 💻 Команды для разработки (Backend)
> **ПРАВИЛО:** Все команды бэкенда выполнять СТРОГО из директории `./backend`[cite: 3]!

*   **Запуск сервера:** `python manage.py runserver`[cite: 3]
*   **Миграции:** `python manage.py makemigrations` и `python manage.py migrate`[cite: 3]
*   **Заполнение БД (Кастомные скрипты):**
    *   Создание тегов по умолчанию: `python manage.py create_default_tags`[cite: 3]
    *   Генерация тестового меню: `python manage.py seed_menu`[cite: 3]

## 💻 Команды для разработки (Frontend)
> **ПРАВИЛО:** Все команды фронтенда выполнять СТРОГО из директории `./frontend`!

*   **Установка зависимостей:** `npm install`
*   **Запуск в dev-режиме:** `npm run dev`
*   **Сборка:** `npm run build`

## 📜 Инженерные соглашения для ИИ-агентов
1. **Изоляция окружения:** Всегда проверяй `pwd` перед выполнением команд в терминале. Переходи в нужную директорию через `cd`.
2. **Синхронизация типов:** При изменении сериализаторов в `backend/core/serializers.py`[cite: 3], ты ОБЯЗАН обновить соответствующие TypeScript-интерфейсы на фронтенде.
3. **Локализация:** При добавлении новых текстовых полей в модели меню, обязательно учитывай их регистрацию в `backend/core/translation.py` и `backend/core/admin.py` для поддержки переводов[cite: 3].
